## ORIENTAÇÕES DO PLANO DE ENSINO

#### Projeto Final

Os alunos formarão duplas para montar um projeto final de maior dificuldade, cobrindo os tópicos vistos em sala de aula.

Os projetos serão propostos pelos alunos, e deverá ser apresentado:
- Um protótipo em funcionamento  
- Um relatório no formato científico (formatação IEEE)

#### Estrutura do Relatório

- **Resumo (1 ponto)**  
  Apresentar o projeto de forma geral e em poucas palavras.

- **Introdução (1 ponto)**  
  Descrição do problema a ser resolvido e revisão bibliográfica do assunto.

- **Desenvolvimento (5 pontos)**  
  Solução com justificativa baseada em fundamentos teóricos. Deve conter:

  - **Descrição do hardware (2 pontos)**  
    - BOM (*Bill of Materials*)  
    - Montagem (diagrama de blocos, esquemático e descrição textual)

  - **Descrição do software (3 pontos)**  
    - Apresentação e justificativa do algoritmo desenvolvido  
    - Explicação textual do funcionamento (fluxogramas ou pseudocódigo)  
    - **Importante:** não consiste em replicar o código  
    - O código deve ser incluído como **apêndice**

- **Resultados (2 pontos)**  
  - Experimentos para validar o projeto  
  - Análise crítica dos resultados esperados e obtidos  
  - Discussão de possíveis causas para divergências

- **Conclusão (1 ponto)**  
  Retomar sucintamente:
  - Problema  
  - Solução  
  - Resultados obtidos

### Pontos de Controle (PC)

Os pontos de controle são prévias do projeto final.

> ⚠️ Cada dia de atraso resulta em **-0,5 pontos** na nota do respectivo ponto de controle.

#### PC1 — Proposta do Projeto
- Apresentação do tema escolhido  
- Definição do problema  
- Pesquisa (produtos existentes, artigos, etc.)  
- Texto contendo:
  - Introdução  
  - Justificativa  
  - Objetivos  
  - Requisitos  
  - Benefícios  
  - Revisão bibliográfica  

----

#### PC2 — Funcionamento Básico
- Conectar partes fundamentais ao Raspberry Pi  
  - Exemplos: GPS, sensores, e-mail, câmera, reconhecimento facial  
- Demonstrar funcionamento individual de cada parte  
- Permitido uso de linguagens interpretadas (Python, Octave, etc.)

**Texto:**
- Descrição de hardware  
- Descrição de software  
- Resultados obtidos  

----

#### PC3 — Refinamento do Protótipo
- Conectar todas as partes fundamentais  
- Acrescentar recursos de sistema:
  - Múltiplos processos e threads  
  - Pipes  
  - Sinais  
  - Semáforos  
  - Mutex  

- Desenvolvimento em **C e/ou C++**

**Texto:**
- Atualização de hardware  
- Atualização de software  
- Resultados  

----

#### PC4 — Refinamento Avançado
- Conectar todas as partes fundamentais  
- Acrescentar recursos avançados:
  - Linux em tempo real  

- Desenvolvimento em **C e/ou C++**

**Texto:**
- Atualização de hardware  
- Atualização de software  
- Resultados  

----

- **PC2:** Disposição das peças  
- **PC3:** Montagem  
- **PC4:** Montagem (refinamento)
